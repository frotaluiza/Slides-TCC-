\begin{slide}{Etapa 2.1: Preparação dos Dados — Padronização}
  \small
  \begin{Item}[0.15]
    \item Padronização Z-score aplicada em \textbf{entradas e saídas} (X e Y)
    \item Ajustada apenas no treino de cada partição (fold)
    \item Evita vazamento de informação entre treino e validação
    \item Garante que todas as variáveis fiquem na mesma escala
    \item $z = \frac{x - \mu}{\sigma}$ — média 0, desvio padrão 1
  \end{Item}
  \note{A padronização Z-score é aplicada em entradas e saídas, ajustada apenas no treino de cada fold para evitar vazamento de informação entre treino e validação. [~1 min]}
\end{slide}

\begin{slide}{Etapa 2.3: Preparação dos Dados — Seleção de \textit{Features} e Discretização}
  \small
  \begin{columns}
    \begin{column}{.5\linewidth}
      \textbf{Seleção de atributos}
      \begin{Item}[0.15]
        \item \textbf{Justificativa:} verificar se o conjunto de 5 preditores selecionados por critério físico poderia ser simplificado sem perda de desempenho
        \item \textbf{Método:} busca exaustiva (\textit{Best Subset Selection}) avaliando todas as $2^5 = 32$ combinações de variáveis com regressão linear multialvo e GroupKFold
        \item \textbf{Resultado:} as configurações com melhor desempenho incluíam todos os 5 preditores — a seleção física inicial foi consistente com a análise baseada em dados
      \end{Item}
    \end{column}
    \begin{column}{.5\linewidth}
      \textbf{Discretização (\textit{Binning})}
      \begin{Item}[0.15]
        \item \textbf{Justificativa:} os experimentos seguem um planejamento fatorial, produzindo regiões de dispersão bem delimitadas para cada variável de entrada — a discretização poderia capturar essas faixas operacionais
        \item \textbf{Método:} mistura Gaussiana unidimensional (critérios AIC/BIC) para definir os intervalos; $2^5 = 32$ configurações contínua vs discretizada avaliadas com regressão linear multialvo
        \item \textbf{Resultado:} nenhuma configuração discretizada apresentou melhoria consistente para os 3 alvos simultaneamente — optou-se por manter a forma contínua original
      \end{Item}
    \end{column}
  \end{columns}
  \note{Tanto a seleção de atributos quanto a discretização foram avaliadas por busca exaustiva com regressão linear multialvo. Nenhuma das estratégias ofereceu ganho consistente para os 3 alvos, confirmando que a seleção física inicial dos preditores e a forma contínua das variáveis eram adequadas. [~1,5 min]}
\end{slide}

\begin{slide}{Etapa 2.4: Preparação dos Dados — Métricas de Erro}
  \small
  \begin{columns}
    \begin{column}{.5\linewidth}
      \textbf{Erro de generalização}
      \begin{Item}[0.15]
        \item Obtido a partir da validação cruzada com GroupKFold
        \item Média do RMSE entre os 3 folds
        \item Mede o desempenho esperado do modelo em regimes não vistos durante o treino
        \item Utilizado durante a busca de hiperparâmetros e seleção de modelos
      \end{Item}
    \end{column}
    \begin{column}{.5\linewidth}
      \textbf{Avaliação final (hold-out)}
      \begin{Item}[0.15]
        \item Conjunto de teste separado \textbf{antes} de qualquer modelagem
        \item 20\% dos dados (35 amostras) com amostras dos 3 regimes de salinidade
        \item O RMSE no hold-out é a métrica de avaliação final para seleção de modelos
        \item Permite estimar o desempenho em condições operacionais representativas
      \end{Item}
    \end{column}
  \end{columns}
  \vfill
  \begin{Item}
    \item O erro de generalização (validação cruzada) guia a busca de hiperparâmetros
    \item O erro de teste (hold-out) é a métrica final para comparar modelos selecionados
  \end{Item}
  \note{Duas métricas são utilizadas: o erro de generalização, obtido pela média do RMSE dos 3 folds do GroupKFold, guia a busca de hiperparâmetros e a seleção dentro de cada família. O erro de teste, calculado no hold-out de 35 amostras separado antes de qualquer modelo, é a métrica final para comparação entre os modelos selecionados. [~1,5 min]}
\end{slide}

\begin{slide}{Etapa 3: Critérios de Seleção}
  \small
  \begin{columns}
    \begin{column}{.5\linewidth}
      \textbf{Critério hierárquico:}
      \begin{Item}[0.15]
        \item \textbf{Nível 1 — Dentro de cada família:} 1-SE + mínima complexidade para escolher hiperparâmetros
        \item \textbf{Nível 2 — Entre famílias:} comparação direta pelo menor RMSE$_{CV}$ no fluxo
      \end{Item}
      \vspace{0.3cm}
      \textbf{Medida de complexidade:} número de parâmetros treináveis
    \end{column}
    \begin{column}{.5\linewidth}
      \textbf{Seleção por target:}
      \begin{Item}[0.15]
        \item Cada variável de saída (Flux, T$_{alim,out}$, T$_{ref,out}$) pode ter um modelo diferente
        \item A busca de hiperparâmetros é otimizada separadamente para cada alvo
        \item Ao final, cada target tem seu próprio modelo selecionado pelo critério 1-SE
      \end{Item}
    \end{column}
  \end{columns}
  \note{O critério de seleção tem dois níveis: dentro de cada família, o 1-SE combinado com mínima complexidade escolhe os melhores hiperparâmetros. Entre famílias, a comparação é direta pelo menor RMSE no fluxo. A seleção por target permite que cada variável de saída tenha o modelo mais adequado. [~1,5 min]}
\end{slide}

      \midrule
      \textbf{Distribuído}
      & Discretiza o domínio espacial; resolve perfis de temperatura e concentração ao longo do canal
      & 1D (difusão molecular + Knudsen), 2D/CFD \\
      \midrule
      \textbf{Reduzido 0D}
      & Parâmetros concentrados; balanços globais de massa e energia; resistências equivalentes
      & Modelo adotado neste trabalho \\

      & Balanços de massa e energia em estado estacionário; sem discretização do domínio
      & Cálculos de fluxo, GOR e SEC \\
      \bottomrule
    \end{tabular}
  }
  \vfill
  \begin{Item}
    \item 3 funções: referência física, comparativo e fonte de informação para híbridos
    \item Código em C executado via Python — gera coluna \texttt{phy}
  \end{Item}
  \note{Três níveis de modelos físicos na literatura de MD. Modelos distribuídos resolvem equações em cada ponto do módulo — alta fidelidade mas alto custo. Modelos reduzidos 0D tratam o módulo como um volume único com grandezas médias — este é o adotado no trabalho. Modelos globais aplicam balanços em estado estacionário sem discretização. O modelo 0D tem 3 funções: referência física, comparativo com data-driven, e fonte de informação para arquiteturas híbridas. [~1,5 min]}
\end{slide}
